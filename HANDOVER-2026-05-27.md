# HCI Prototype Interface — Handover (2026-05-27)

처음 코드를 받아보는 사람이 30분 안에 전체 그림을 파악할 수 있도록 정리한 문서입니다.

---

## 1. 프로젝트 개요

- **무엇**: 완전자율주행(Lv5) 차량의 16" 인포테인먼트 HMI 프로토타입 + 연구자용 오퍼레이터 콘솔
- **목적**: 정적 사용성 평가용 디자인 프로토타입. AI/TTS/STT 등 음성 기능은 **없음** (자매 프로젝트 `HCI-prototype`에는 있음). 이 프로젝트는 **디자인과 인터랙션이 더 풍부한 전통적 네비게이션** 중심
- **레퍼런스**: 자매 프로젝트 `c:\Users\User\Desktop\Cursor\HCI-prototype` — 이 프로젝트의 코드 일부는 거기서 가져온 것 (디자인 토큰, 5종 앱, 시나리오, 오퍼레이터 콘솔 구조)
- **목표 해상도**: 1920×1080 (`.hmi-viewport` 가 다른 비율에선 letterbox 처리)

## 3. 실행

```bash
# 1) 의존성 설치 (최초 한 번)
npm install

# 2) 카카오맵 키 설정 (선택)
cp .env.example .env.local
# .env.local 의 VITE_KAKAO_JS_KEY 를 본인의 JS 키로 교체
# (https://developers.kakao.com → 내 애플리케이션 → 앱 키 → JavaScript)
# 키가 없으면 NavigationApp 풀스크린이 "키 미설정" placeholder를 표시함.
# 그 외 모든 기능은 정상 동작.

# 3) 개발 서버
npm run dev
# → http://localhost:5173/         (= /hmi, 운전자 화면)
# → http://localhost:5173/operator (오퍼레이터 콘솔, 별 탭에서 열면 BroadcastChannel로 자동 동기화)

# 4) 빌드
npm run build      # → dist/
npm run preview    # 빌드 산출물 로컬 미리보기
```

## 6. 라우팅 구조

[src/App.jsx](src/App.jsx) 맨 아래에 라우터 셸이 정의되어 있습니다.

```
<BrowserRouter>
  <ExperimentProvider>
    /            → <VehicleHMI />     (참가자용 차량 HMI)
    /hmi         → <VehicleHMI />     (동일, 별칭)
    /operator    → <OperatorConsole /> (연구자 콘솔)
    *            → Navigate /
  </ExperimentProvider>
</BrowserRouter>
```

**중요**: `/operator` 는 새 탭에서 열어야 함. 같은 origin이라면 BroadcastChannel(`exp_channel`)이 두 탭의 상태를 자동 동기화함. ExperimentContext의 `useEffect`(lines 65-150)가 채널을 구독함.

## 7. 상태 관리

### 7-1. React state (`App.jsx` 내부 — VehicleHMI 컴포넌트)
| 상태 | 용도 |
|---|---|
| `activeApp` | 현재 열린 앱 (`'Music'`/`'Mail'`/`'Phone'`/`'Calendar'`/`'Navigation'`/`null`) |
| `isMusicFullscreen` 등 5개 | 각 앱 풀스크린 토글 |
| `isBriefingOpen` | 우측 BriefingPanel 표시 여부 |
| `simStage`, `simType` | 시뮬레이션 단계 (`idle` / `attempting` / `aquaplaning_active` / ...) |
| `alertConfig` | 상단 알림 토스트 콘텐츠 |
| `temperature`, `isAutoClimate`, `fanSpeed` | 공조 |
| `volume`, `muted`, `volumeOpen` | 시스템 볼륨 (자동 3초 후 collapse) |
| `currentSpeed`, `currentTime` | 속도/시계 (시계는 1초 tick) |
| **`activeRoute`** | **NavigationApp이 `setActiveRoute` 호출 시 채워짐. 좌측 위젯이 이를 보고 경로 렌더링** |

### 7-2. ExperimentContext (`src/context/ExperimentContext.jsx`)
실험 / 오퍼레이터 콘솔용 글로벌 상태. **단일 ExperimentProvider가 HMI와 OperatorConsole을 둘 다 감쌈**.

| 상태 | 의미 |
|---|---|
| `currentParticipant` | `{ participantId, ageRange, drivingExperience, dev }` |
| `currentTrial` | 현재 진행 중인 시험 객체 |
| `liveConversationTurns` | (현재 항상 빈 배열 — 채팅 미구현) |
| `experimentPhase` | `'setup'` → `'trial_active'` → `'review'` → `'saved'` |
| `activeScenario` | 시나리오 객체 (Alt+Q/W 또는 콘솔에서 변경) |
| `saveStatus` | `{ autosaved, finalSaved, exported }` |
| `hmiResetNonce` | 콘솔이 HMI를 리셋시키고 싶을 때 bump됨. App.jsx의 effect가 이를 감지해 simStage/앱들을 리셋 |

**Exposed actions**: `startTrial`, `endTrial`, `doFinalSave`, `setScenario`, `resetHmi`, `initializeHMI`, `discardSession`, `addAnotherTrial`, `startNewParticipant`, `markExported`

### 7-3. BroadcastChannel 메시지 (`exp_channel`)
| 메시지 | 송신 | 동작 |
|---|---|---|
| `START_TRIAL` | Operator → HMI | 시나리오 활성화, phase=`trial_active` |
| `END_TRIAL` | Operator → HMI | phase=`review`, HMI 리셋 |
| `INITIALIZE_HMI` | Operator → HMI | 모든 상태 초기화 |
| `SET_SCENARIO` | Operator ↔ HMI | activeScenario 변경 (Alt+Q/W) |
| `RESET_HMI` | Operator ↔ HMI | HMI를 idle로 (Alt+R) |
| `TURN_CREATED`/`TURN_UPDATED` | HMI → Operator | (현재 미사용 — 채팅 없음) |

### 7-4. 로컬 영속 (localStorage)
`sessionLogger.js`의 키들:
- `exp_participant_counter` — 다음 참가자 ID 카운터
- `exp_trial_counter_<participantId>` — 참가자별 시험 카운터
- `exp_current_participant` / `exp_current_trial` / `exp_current_turns` — 새로고침 복구
- `exp_saved_sessions` — 최종 저장된 참가자/시험 데이터 누적

JSON / Markdown export는 모두 `sessionLogger`의 함수로 다운로드.

## 8. 데이터 흐름: NavigationApp → 좌측 위젯

```
[NavigationApp]
  사용자가 PLACES에서 목적지 클릭
    ↓
  setView('route') + setDestination(place)
    ↓
  useEffect → fetchDrivingRoute(origin, destination)  ← OSRM 호출
    ↓
  setPreviewRoute({ distance, duration, geometry })
    ↓
  추천/최단/무료 카드에 실측 distance/duration 표시
    ↓
  사용자가 "안내 시작" 클릭
    ↓
  confirmRoute() → setActiveRoute({          ← prop으로 받은 setter (App.jsx의 state)
    destination, geometry, distanceM,
    durationSec, departureIso, baseArrivalIso
  })
    + onClose()                              ← NavigationApp 닫힘

[App.jsx — VehicleHMI]
  activeRoute 가 set됨 + 30s tick interval 시작
    ↓
  좌측 위젯 IIFE에서 buildRouteSvg(geometry, 297, 135) 로 SVG path 생성
    ↓
  잔여거리 = distanceM × (1 - elapsedSec / durationSec)
  잔여시간 = (baseArrivalIso - now) / 60_000
  도착예정 = formatClockTime(new Date(baseArrivalIso))
    ↓
  30초마다 routeTick이 bump → 리렌더 → 위 값들이 현재 시간 기준으로 갱신
```

**해제**: NavigationApp 사이드바의 "안내 종료" 링크 → `setActiveRoute(null)` → 위젯이 idle 상태("어디로 갈까요?")로 돌아감.

## 12. 카카오 맵 + OSRM 라우팅

[src/utils/kakaoNav.js](src/utils/kakaoNav.js)

### Kakao Maps JS SDK
- `loadKakaoSdk(key)` — Idempotent 동적 로더. 최초 한 번만 `<script>` 주입, `services` 라이브러리 포함
- `services` 라이브러리는 `kakao.maps.services.Places` (현재는 미사용 — 검색은 정적 PLACES 리스트)
- `KAKAO_JS_KEY` 미설정 시 fallback UI 표시 (앱은 정상 동작)

### OSRM (경로 계산)
- `https://router.project-osrm.org/route/v1/driving/{origin};{dest}?geometries=geojson&overview=full`
- 공개 데모 서버 — 무료, 인증 불필요, CORS 허용
- 응답: `{ distance(m), duration(sec), geometry: [[lng,lat], ...] }`
- **카카오 directions API는 사용하지 않음** (별도 인증 + 비용 발생)

### 좌표 (`DEFAULT_CENTER`)
모든 경로의 origin은 고정 = 홍익대학교 (`37.5510, 126.9251`). 차량이 실제로 움직이지 않는 정적 프로토타입이라 origin을 한 곳에 고정.

## 14. 5종 앱 패턴 (`isFullscreen` / `isBriefingOpen` props)

5개 앱 모두 동일한 외부 래퍼 패턴을 따름. 새 앱 추가 시 이걸 그대로 베끼면 됨.

```jsx
export default function FooApp({ isFullscreen, setIsFullscreen, isBriefingOpen, onClose }) {
  // 폭/위치 계산: briefing 패널이 열려 있으면 1410 - 466, 풀스크린이면 1920(- 466)
  const briefingWidth = 420 + 30 + 16
  const getWidth = () => {
    if (isFullscreen && isBriefingOpen) return 1920 - briefingWidth
    if (isFullscreen) return 1920
    if (isBriefingOpen) return 1410 - briefingWidth
    return 1410
  }
  const getLeft = () => (isFullscreen ? 0 : 461)

  return (
    <motion.div
      initial={{ opacity: 0, y: 60, left: getLeft(), top: 104, width: getWidth(), height: 830 }}
      animate={{ opacity: 1, y: 0, width: getWidth(), height: isFullscreen ? 880 : 830,
                 left: getLeft(), top: isFullscreen ? 79 : 104 }}
      exit={{ opacity: 0, y: 60 }}
      transition={{ type: 'spring', stiffness: 250, damping: 30 }}
      className="absolute z-30"
      style={{ borderRadius: isFullscreen ? '0px' : '32px', ... }}
    >
      {/* 우상단: [풀스크린 토글] [닫기] */}
      {/* 본문 */}
    </motion.div>
  )
}
```

App.jsx 쪽 마운트:
```jsx
{activeApp === 'Foo' && (
  <FooApp
    isFullscreen={isFooFullscreen}
    setIsFullscreen={setIsFooFullscreen}
    isBriefingOpen={isBriefingOpen}
    onClose={() => { setActiveApp(null); setIsFooFullscreen(false) }}
  />
)}
```
